# stt-cursor

Непрерывная оффлайн-диктовка для Linux, которая захватывает звук с микрофона и вставляет распознанный текст прямо в позицию курсора. 

В новой версии проект переведён с Vosk на **Google SODA** (Speech On-Device API из ChromeOS/Android) через фреймворк [gasr](https://github.com/biemster/gasr). Это даёт **автопунктуацию, автоматические заглавные буквы и преобразование чисел прямо из коробки**.

## Оборудование и среда тестирования
Проект успешно протестирован на следующей конфигурации:
* **ОС:** Arch Linux (Kernel 6.19.10-arch1-1)
* **DE:** Xfce 4.20
* **Оборудование:** AMD Ryzen 5 5625U (Radeon Graphics)

## Как это работает
1. `parec` (PulseAudio/PipeWire) захватывает аудиопоток с микрофона (16 кГц, моно).
2. `gasr.py` и `libsoda.so` выполняют распознавание речи в реальном времени.
3. Python-скрипт обрабатывает результаты: фильтрует явные голосовые команды (например, «новая строка» или «собака») и управляет буфером обмена.
4. `xclip` копирует итоговый текст, а `xdotool` эмулирует нажатие `Ctrl+V` (или `Ctrl+Shift+V` в терминалах) для вставки текста.

## Системные зависимости

Убедитесь, что в вашей системе установлены необходимые пакеты. Сторонние зависимости Python больше не требуются (`uv` можно не использовать, достаточно стандартного `python3`).

Для Arch Linux:
```bash
sudo pacman -S xdotool xclip libnotify pipewire-pulse libc++ alsa-utils git python
```

| Пакет | Назначение |
|---|---|
| `pipewire-pulse` | `parec` — захват аудио с микрофона |
| `xclip`, `xdotool` | Управление буфером обмена и эмуляция вставки (X11) |
| `libnotify` | Системные уведомления (`notify-send`) |
| `libc++` | **Необходимо** для работы бинарной библиотеки `libsoda.so` |
| `alsa-utils` | Рекомендуется для корректной работы аудиоподсистемы |

## Установка

Все необходимые скрипты, модели и библиотеки загружаются и настраиваются автоматически.

```bash
# 1. Клонируем репозиторий
git clone https://codeberg.org/femto/stt-cursor.git
# или: 
# git clone git@github.com:AnDsergey13/stt-cursor.git
cd stt-cursor

# 2. Выдаем права на исполнение
chmod +x setup.sh dictation-toggle.sh

# 3. Запускаем автоматическую настройку
./setup.sh
```

**Что делает `setup.sh`:**
* Клонирует движок `gasr`.
* Загружает бинарный файл `libsoda.so`.
* Загружает языковые/ASR модели (`ru-ru`, `en-us`, `zork`).
* Автоматически устанавливает русскую модель по умолчанию (через симлинк `SODAModels`).

## Глобальная горячая клавиша (рекомендуется)

В комплекте идет скрипт `dictation-toggle.sh`. Он автоматически определяет директорию проекта (вам **больше не нужно** вручную редактировать пути в файле) и переключает диктовку одной клавишей.

### Настройка в Xfce:
1. Откройте: **Настройки → Клавиатура → Комбинации клавиш приложений → Добавить**
2. В поле **Команда** укажите полный абсолютный путь к скрипту, например:
   `/home/ИМЯ_ПОЛЬЗОВАТЕЛЯ/path/to/stt-cursor/dictation-toggle.sh`
   *(Внимание: Xfce не понимает символ `~`, используйте абсолютный путь `/home/...`)*
3. Назначьте удобную комбинацию (например, `Alt+X`).

**Как это работает:**
* Первое нажатие запускает диктовку (появится уведомление «🎙 Диктовка: SODA запущена»).
* Повторное нажатие корректно завершает процессы и очищает память (уведомление «🎙 Диктовка: Остановлена»).

## Голосовая пунктуация

SODA автоматически расставляет базовые точки, запятые и капитализирует текст. Однако скрипт поддерживает явные голосовые команды для точного форматирования:

| Вы произносите | Результат |
|---|:---:|
| точка | `.` |
| запятая | `,` |
| вопросительный знак | `?` |
| восклицательный знак | `!` |
| точка с запятой | `;` |
| двоеточие | `:` |
| тире | ` — ` |
| дефис | `-` |
| новая строка | `\n` |
| новый абзац | `\n\n` |
| открыть скобку | `(` |
| закрыть скобку | `)` |

*Скрипт также автоматически удаляет лишние пробелы перед знаками препинания и после открывающих скобок.*

## Смена языка распознавания (на Английский)

По умолчанию `setup.sh` делает симлинк на русскую модель (которая отлично справляется со смесью русского и английского языков). Если вам нужна чисто английская модель:

```bash
cd gasr/
rm SODAModels
ln -s SODAModels_en-us SODAModels
```

## Ограничения и Troubleshooting

* **Только X11**: Wayland пока не поддерживается (ограничение эмуляции через `xdotool`).
* **Оффлайн**: Интернет требуется только один раз во время выполнения `setup.sh`.
* **Терминалы**: Скрипт распознает работу в терминале (`xfce4-terminal`, `alacritty`, `kitty` и др.) и автоматически использует комбинацию `Ctrl+Shift+V` для вставки без багов.

**Частые проблемы:**
* **✗ Не найден gasr.py:** Вы не запустили `./setup.sh` перед стартом программы.
* **✗ parec не найден:** Убедитесь, что установлен пакет `pipewire-pulse` (или `pulseaudio-utils`).
* **Не запускается / Ошибка libsoda.so:** Проверьте наличие пакета `libc++` (`sudo pacman -S libc++`).
