# stt-cursor

Непрерывная оффлайн-диктовка для Linux (Arch Linux): захват микрофона → распознавание речи (Vosk) → вставка текста в позицию курсора.

## Как это работает

1. `parec` (PulseAudio/PipeWire) захватывает аудиопоток с микрофона в формате `s16le`, 16 кГц, моно
2. Vosk (`KaldiRecognizer`) распознаёт речь в реальном времени
3. Распознанный текст проходит постобработку (пунктуация голосом, капитализация)
4. `xclip` кладёт текст в буфер обмена, `xdotool`(❗️ не проверено❗️) эмулирует `Ctrl+V`

## Системные зависимости

```bash
sudo pacman -S xdotool xclip libnotify pipewire-pulse
```

| Пакет | Зачем |
|---|---|
| `pipewire-pulse` | `parec` — захват аудио с микрофона |
| `xclip` | вставка текста через буфер обмена (надёжно для кириллицы) |
| `xdotool` | эмуляция `Ctrl+V` (только X11) |
| `libnotify` | уведомления через `notify-send` |

## Установка

```bash
git clone ssh://git@codeberg.org/femto/stt-cursor.git
cd stt-cursor

echo "3.14.2" > .python-version
uv init
uv venv --python 3.14.2
uv add vosk

# Скачать модель распознавания речи (~45 МБ)
wget alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip
unzip vosk-model-small-ru-0.22.zip
mv vosk-model-small-ru-0.22 model
rm vosk-model-small-ru-0.22.zip
```

### Альтернативные модели

- **Большая модель** (~1.8 ГБ, выше точность): `wget alphacephei.com/kaldi/models/vosk-model-ru-0.42.zip`
- Маленькая модель потребляет ~300 МБ RAM, большая — 4–8 ГБ.

## Горячая клавиша (toggle-скрипт)

Скрипт `dictation-toggle.sh` из репозитория запускает и останавливает диктовку одной клавишей. Чтобы подключить его:

### 1. Отредактируйте путь в скрипте

Откройте `dictation-toggle.sh` и убедитесь, что переменная `PROJECT` указывает на корень этого репозитория:

```bash
PROJECT="$HOME/your-path"  # ← ваш реальный путь
```

Проверить путь: `cd` в папку проекта и выполните `pwd`.

### 2. Скопируйте скрипт в PATH

```bash
cp dictation-toggle.sh ~/.local/bin/
chmod +x ~/.local/bin/dictation-toggle.sh
```

### 3. Назначьте горячую клавишу в Xfce

**Настройки → Клавиатура → Комбинации клавиш приложений → Добавить**

- **Команда** — полный абсолютный путь:
  ```
  /home/USER/.local/bin/dictation-toggle.sh
  ```
- **Клавиша** — например, `Alt+X`

> ⚠️ **XFCE не раскрывает `~`** — писать `~/.local/bin/...` нельзя, только полный путь от `/home/`.

### Как работает

- Первое нажатие → запуск диктовки + уведомление «Запущена»
- Повторное нажатие → остановка + уведомление «Остановлена»
- PID процесса: `/tmp/stt-cursor.pid`
- Лог: `/tmp/stt-cursor.log`

## Голосовая пунктуация

Произносите команды для расстановки знаков препинания:

| Голосовая команда | Результат |
|---|---|
| точка | `.` |
| запятая | `,` |
| вопросительный знак | `?` |
| восклицательный знак | `!` |
| новая строка | `\n` |
| двоеточие | `:` |
| тире | ` — ` |

Первая буква после знака конца предложения автоматически капитализируется.

## Ограничения

- Только **X11** — Wayland не поддерживается.
- Полностью **оффлайн** — интернет не нужен.
- Эмуляция `Ctrl+V` через xdotool, не проверена.
- Текст от Vosk приходит без пунктуации — знаки расставляются голосовыми командами.
- Первый запуск может иметь задержку на загрузку модели.

## Troubleshooting

| Проблема | Решение |
|---|---|
| `Failed to create a model` | Проверьте, что `model/` содержит `am/`, `conf/`, `graph/`, `ivector/` (без вложенной папки) |
| Toggle не работает по горячей клавише | Команда в Xfce должна быть полным путём без `~` |
| Уведомления не появляются | `xfconf-query -c xfce4-notifyd -p /do-not-disturb` — если `true`, поставьте `false` |
| `parec` не захватывает звук | `pactl list short sources` — проверьте, что микрофон не замьючен |
| `No such file or directory` | Переменная `PROJECT` в скрипте не совпадает с реальным путём |
